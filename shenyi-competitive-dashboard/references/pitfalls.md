## 1. 数据提取

### 把缺失值当零

`-`、空白和无公式缓存值通常表示未知，不是零。统一保存为 `null`；只有明确数字 `0` 才保存精确零。

### 静默吞掉解析失败

未知格式返回 `None` 会让真实错误伪装成缺失。非空未知值应报错，并记录文件、Sheet、行号、字段和原值。

### 正则只匹配前缀

`re.match` 可能把带尾随文本的脏值当成成功。数值解析使用完整匹配，并在提取后检查所有指标中是否残留字符串。

### 商品 ID 污染

商品 ID 可能含前导单引号或被读成科学计数法。清洗后必须为纯数字；无法还原完整 ID 时停止生成跳转链接。

### 日期系统和窗口混淆

检查工作簿的 1900/1904 epoch。自然月、Excel 日期和滚动窗口分别规范化；不要因为起止日期接近就自动映射。

### 依赖固定列顺序

导出版本变化后列位置可能变化。按规范化表头映射，并对重复表头、缺失字段和未知必需字段报错。

### 重复行直接求和

重复可能是重复导出，而非应相加的分片。先按业务键列出重复，再由文件语义决定去重或聚合。

## 2. 数据合并

### 使用“竞品1”跨文件匹配

类别编号只在单次导出中有意义。跨文件以清洗后的商品 ID、稳定店铺 ID 和用户确认的竞品映射为准。

### 全局日期取第一个对象

第一个对象可能缺期。全局日期必须是所有对象日期并集；每个指标按日期键补 `null` 对齐。

### Top1 观察期不一致

累计值会偏向数据期数较多的链接。Top1 只在共同窗口排名，并把排名期写入说明。

### 流量格式半迁移

店铺流量必须使用 `{top:{}, l1:{}, l2:{}}` 对象格式，单品流量必须使用记录数组。不要混用。

### 必需字段缺失

竞品级检查 `price`、`price_note`、`price_tier`、`all_link_ids`、`links`；链接级检查商品信息、日期、五项指标、累计买家、关键词、SKU、流量；店铺级检查类别、日期和五项指标。

## 3. 分析口径

### GMV 被写成真实成交额

买家数乘参考价没有考虑件单量、优惠、退款和真实成交价。标题、tooltip 和说明统一写“GMV 估算”。缺少用户确认的参考价时不计算。

### 官方转化率与自行计算混用

脱敏后的访客、买家、转化率可能不代数一致。保留官方字段；自行计算值必须换名并说明。

### SKU 跨窗口反算

只有 SKU 占比与总买家数属于同一商品链接、同一期间时才反算人数。否则只展示占比或标为不可比。

### 策略标签缺少证据

“付费驱动”“自然流量强”等标签必须附指标、期间和阈值。不要把同步变化写成因果关系。

## 4. 前端实现

### 用户数据进入可执行 HTML

商品名或搜索词可能包含 HTML。数据放入 `application/json`，序列化时把 `<` 转义为 `\u003c`；动态文本用 `textContent`，不生成内联 `onclick`。

### 渲染目标不存在或 ID 重复

模板 ID 与渲染代码可能漂移。检查 `getElementById`/选择器目标存在，并检查静态 ID 唯一。

### 估值切换漏模块

把模块渲染器注册到数组，由 Tab 入口统一调用，减少手工触发链遗漏。

### Chart.js 实例未销毁

重绘前销毁旧实例。重复切换 Tab 和估值，确认无重叠、无“canvas already in use”错误。

### 调色板轮换导致对象变色

颜色按稳定对象键映射，不要按筛选后的临时索引重新分配。颜色数不足时扩充，而不是静默循环。

### JSON 替换覆盖脚本

不要用贪婪正则替换整个 `<script>`。优先使用独立 `application/json` 数据块和明确边界；替换后执行 JS 与浏览器回归。
