## 1. 数据性质与审计要求

数据通常来自“生意参谋 → 竞争 → 竞品分析”。竞店和竞品指标多为平台脱敏后的区间，不是精确事实。

处理前逐 Sheet 记录：真实表头、行数、字段类型、空值、数值格式样例、最早/最晚日期、店铺数、商品数和重复业务键。不要依赖固定列序；按规范化后的表头映射字段，并对缺失必需字段报错。

用 `openpyxl.load_workbook(path, data_only=True, read_only=True)` 读取。若公式单元格没有缓存结果，要求用户先用表格软件重算并保存，或改用可计算公式的工具；不能把 `None` 当零。

## 2. 区间值与缺失值

### 2.1 规范结构

```json
{"low": 1000, "mid": 1750, "high": 2500}
{"low": 5, "mid": 5, "high": 5, "exact": true}
```

必须满足 `low <= mid <= high`。百分比按小数存储，例如 `1%` 存为 `0.01`。

### 2.2 常见格式

| 原值 | 结果 |
|---|---|
| `1000 ~ 2500` | 1000 / 1750 / 2500 |
| `1万 ～ 2.5万` | 10000 / 17500 / 25000 |
| `0% 至 1%` | 0 / 0.005 / 0.01 |
| `5` | 5 / 5 / 5，精确值 |
| `0` | 0 / 0 / 0，精确值 |

解析器应接受常见千分位、半角/全角波浪号、`至` 和 `万/亿`单位，但必须使用完整匹配，不能只解析字符串前半段。

### 2.3 缺失不是零

`None`、空白、`-`、`--`、`N/A` 表示缺失，规范化为 `null`。非空且不属于已知格式的值属于解析失败，必须记录位置并终止数据构建。

如实际导出文件明确将某个符号定义为零，可按文件级规则覆盖，但要写入数据审计摘要。

### 2.4 原始值追溯

规范化 JSON 可以只保存计算值，但提取过程必须保留解析错误清单，包含文件、Sheet、Excel 行号、字段、原值。商品 ID 同时保留原值和清洗值，便于定位问题。

## 3. 日期与比较窗口

可能出现：

- Excel 日期对象
- Excel 1900 日期系统序列号
- 标准日期字符串 `2026-02-01`
- 自然月范围 `2026-02-01 ~ 2026-02-28`
- 滚动 30 天范围 `2026-04-23 ~ 2026-05-22`
- 月份标签 `2月`

不要给已经包含“月”的标签再次追加后缀。

Excel 1900 日期系统可用 `1899-12-30 + serial` 转换；读取前检查工作簿 epoch，不能默认所有文件都使用 1900 系统。

用规范日期键建立日期并集，并按键对齐每个对象。缺失期填 `null`，不能用相邻期数据填补。

自然月和滚动窗口不是同一统计口径。不同滚动窗口即使相差两天也不自动合并；用户确认后可以映射到展示标签，但原始起止日期必须保留并在模块说明中披露。

## 4. 商品 ID 与业务键

商品 ID 可能带前导单引号。清洗时只保留数字：

```python
re.sub(r"[^0-9]", "", str(value))
```

清洗结果为空时报错。跳转链接使用：

```text
https://item.taobao.com/item.htm?id={clean_product_id}
```

推荐业务键：

- 店铺月度：`store_id 或 store_name + period_key`
- 店铺关键词：`store + period_key + term`
- 店铺流量：`store + period_key + source + level1 + level2`
- 单品月度：`product_id + period_key`
- 单品关键词：`product_id + period_key + term`
- SKU：`product_id + period_key + sku`

重复业务键必须审计。只有确认是重复导出行后才能去重；不能直接求和。

## 5. 流量层级

原始行通常由 `流量来源`、`一级来源`、`二级来源` 表达层级：

- 顶层：`一级来源`为空
- 一级：`一级来源`有值，`二级来源`为空
- 二级：两者都有值

单品流量使用数组结构：

```json
[
  {"source":"经营优势","level1":null,"level2":null,"visitors":{"low":1,"mid":2,"high":3}},
  {"source":"搜索","level1":"经营优势","level2":null,"visitors":{"low":1,"mid":2,"high":3}},
  {"source":"关键词推广","level1":"付费推广","level2":"无界","visitors":{"low":1,"mid":2,"high":3}}
]
```

店铺流量使用 `{top:{}, l1:{}, l2:{}}` 对象结构，单品流量使用上述数组结构。提取时按分析对象生成对应格式，不增加额外迁移层。

## 6. Sheet 语义

常见 Sheet 名称可能变化，按表头和内容确认：

| 业务表 | 典型字段 | 粒度 |
|---|---|---|
| 店铺数据 | 店铺、日期、访客、买家、转化率、加购、收藏 | 店铺 × 期间 |
| 店铺关键词 | 期间、店铺、搜索词、访客、买家 | 店铺 × 期间 × 词 |
| 店铺流量 | 期间、店铺、来源层级、访客、买家、转化率 | 店铺 × 期间 × 来源 |
| 单品数据 | 商品、商品 ID、日期及核心指标 | 商品链接 × 期间 |
| 单品关键词 | 期间、商品 ID、搜索词及指标 | 商品链接 × 期间 × 词 |
| 单品流量 | 期间、商品 ID、来源层级及指标 | 商品链接 × 期间 × 来源 |
| SKU 占比 | 期间、SKU、商品 ID、支付买家数占比 | 商品链接 × 期间 × SKU |

SKU 表通常只有占比。估算人数时使用“同链接、同期间支付买家数 × SKU 占比”，并标注为估算；找不到完全一致的期间时不计算。

用户自建的竞品清单或品类表不属于固定结构。先识别其用途，再决定是否作为配置或补充数据，不要按标准 Sheet 强行解析。

## 7. 多文件合并

1. 以清洗后的商品 ID 和稳定店铺 ID 优先匹配，不依赖“竞品1”等临时类别编号。
2. 对同名店铺、改名店铺和重复商品链接输出冲突清单。
3. 日期先保留原始窗口，再决定展示对齐方式。
4. 只有业务键和统计窗口都一致时才允许合并或去重。
5. Top1 链接排名使用同一观察窗口内的累计买家中值；窗口不一致时分开排名或先确认共同窗口。

## 8. 分析口径限制

- 平台给出的访客、买家和转化率可能分别脱敏，三者不一定代数一致。
- 区间中值是展示假设，不代表期望值或真实值。
- `参考价 × 支付买家数` 忽略件单量、优惠、退款和实际成交价，只能称为 GMV 估算。
- 搜索词、流量结构和 SKU 可能只覆盖 Top 数据，不能默认占比合计为 100%。
- 策略标签必须给出对应指标与期间；避免把同步变化写成因果关系。
